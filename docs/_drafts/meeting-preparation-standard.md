開會開久了，發現多數卡住的會議，卡法都差不多。我認為開會是有遊戲規則的，掌握好就可以玩出有效率的會議。

<!-- more -->

舉個例子：一群朋友討論出遊計畫，A 把交通、住宿、景點都查好了，進來就開始說哪天出發、住哪比較划算。但 B 和 C 只知道「要去旅遊」，什麼都沒查。這樣的狀態進到「決定行程」的討論，本來就很難往前走。

後來想想，問題不在 B 和 C 沒準備，而是一開始就沒有人說清楚這次見面的目的是「把住宿跟交通定下來」。他們知道要討論旅遊，但沒有人告訴他們要帶著查好的資料來、這次要做決定的。沒有說明 why，對方就不知道要準備什麼。

## 開會前的基本準備

會議約好了就要排到行事曆上。這個基本習慣很容易被忽略，但沒做到的話，與會者不一定知道這件事。

除了時間，開會前還需要：

- 定義會議目的是什麼
- 提前告知每個人可能需要做什麼準備
- 相關文件提前看過

這些事情沒做，會議自然就很容易卡。

## 主持人的角色

主持人不需要事先知道每一個細節，一定有他不懂的領域。

主持人的功能是掌握會議流程、確認是否達到目的、協調發言、幫大家知道現在有沒有結論了。遇到不懂的細節，可以追問負責的人，透過他的回答來抓到答案，不是主持人要自己生答案。

## 與會者的角色

與會者要自行把關自己需要準備的東西。相關文件提前看過，進會議後直接針對問題討論，不是進來之後才開始理解這場會議要做什麼。

這件事不應該由召集人來把關，他不一定清楚每個人的領域要準備什麼。核心問題不是誰負責確認別人有沒有準備好，而是一開始就沒有說清楚這場會議的目的跟大家需要準備什麼。

## 好的會議長什麼樣

大家在進會議之前就已經讀完會議資料。進到會議後直接針對問題討論，不會花時間重新理解這個會議要做什麼。

最後有結論，可能是一個決策、下一步要做什麼，或是提前約好下次會議要交付的東西跟要討論什麼。

這才算會議結束了，不只是時間到了散會。
